Accessibility

Accessibility is an essential part of the modern website design. Creating useful websites with lasting designs means taking every potential user in mind. Disability is present in everyone's lives at some point, so designing for the "outliers" will essentially be designing for everyone.

Accessable Desing

In designing a website, in regards to accessibility, the following choices are important:

  • Provide the page-reader with useful "alt" text-descriptions.
  • Make sure the page's style doesn't interfere with visual impairments, like low contrasts on text and useful features.
  • Make sure buttons and other interactable features are easy to distignuish and use.

Accessibility Test

The Topical page was run through the automated accessibility test with the use of Wave-test.

As the pictures below shows, the test came back with one error, and four alerts. The test was conducted while the page was 90% finished, so that the test could give important feedback.

Under the details of the wave-test, we are given the reason behind the one error and four alerts. The error an empty form label, refering to the dropdown-menu on phone-screens.

The four alerts refere to the fact that all the pictures on the page have identical alternative descriptions for screen-readers etc. This stems from the fact that the page was lacking it's last finish, and every picture's alt= was filled with a generic filler-text. The test did therefore help us to remember this before the site is "shipped."

Using the feedback

After having used the feecback from the Wave test, we had no more alerts. As shown in the picture below.

But automated tests like Wave test are limited in their scope, and so common practices play an important part in designing for Accessibility.
